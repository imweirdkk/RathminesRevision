# Common obstacles in software requirements gathering

- Unclear or vague requirements
	- A stakeholder says: "The system should be fast", "fast" isn't measurable so it's unclear 
- Poor communication between stakeholders
	- Leads to conflicting requirements 
- Changing Requirements (Scope Creep)
	- After development starts, the client requests new features not in the original scope which leads to increased cost and project delays
- Lack of Stakeholder Involvement
	- Impact: Missing critical requirements; Delayed decisions; Incorrect assumptions by the project team 
- Conflicting requirements from different stakeholders
	- Finance teams wants strict approval workflows
	- Operations team wants faster approval; unable to make both happy
- Time constraints
- Unrealistic expectation 
- Inadequate documentation
